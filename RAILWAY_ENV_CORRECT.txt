KEY DIFFERENCE FROM PREVIOUS VERSION:
===========================================

OLD (doesn't work with Railway):
DATABASE_URL=postgresql+asyncpg://postgres:[PASSWORD]@db.[PROJECT_ID].supabase.co:5432/postgres

NEW (works with Railway):
DATABASE_URL=postgresql+asyncpg://postgres:[PASSWORD]@db.[PROJECT_ID]-pooler.supabase.co:5432/postgres
                                                                       ^^^^^^^
                                                                       Notice the -pooler suffix!

===========================================
HOW TO UPDATE IN RAILWAY:
===========================================

1. Go to Railway dashboard
2. Click "Variables" tab
3. Find DATABASE_URL
4. Replace with the new URL (with -pooler)
5. Railway will auto-redeploy
6. Check logs - should see "Application startup complete"

===========================================
IMPORTANT NOTES:
===========================================

- Replace all [YOUR_*] placeholders with your actual values
- The real values are in RAILWAY_ENV_COMPLETE.txt (local file, not in git)
- Never commit actual API keys to git
- Use the values from your local environment files

===========================================
WHY THIS MATTERS:
===========================================

- Direct connection = meant for long-lived local dev
- Pooler connection = meant for Railway/serverless
- Fixes "Network unreachable" error
- Much more reliable for container deployments
- Prevents connection limit issues
